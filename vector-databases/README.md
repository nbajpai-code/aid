# 📂 Vector Databases & Retrieval

Vector databases and indexing libraries are designed to store, index, and query high-dimensional vector representations (embeddings) generated by machine learning models. They form the core infrastructure for Retrieval-Augmented Generation (RAG).

---

## 🗺️ Vector Storage Comparison

Here is a quick overview of when to use which vector storage solution:

| Database | Type | Written In | Core Strength |
| :--- | :--- | :--- | :--- |
| **Qdrant** | Standalone / Cloud | Rust | Extremely fast, rich metadata payload filtering, highly optimized memory footprint. |
| **Pinecone** | Serverless / SaaS | C++ / Go | Fully managed, zero infrastructure overhead, handles massive scale easily. |
| **Milvus** | Distributed Cluster | Go / C++ | Cloud-native, built for multi-billion scale enterprise environments. |
| **Chroma** | Embedded / Local | Python / C++ | Zero-config, perfect for quick prototyping and local Python projects. |
| **Weaviate** | Standalone / Cloud | Go | Hybrid search (sparse + dense), vectorizes data automatically, GraphQL API. |
| **pgvector** | PostgreSQL Extension | C | Store vector embeddings directly alongside your relational database columns. |
| **Faiss** | Core Index Library | C++ / Python | Absolute lowest latency for raw vector search in memory. Developed by Meta. |
| **LanceDB** | Serverless / Embedded | Rust / Python | Native multimodal support, stores data in highly efficient Lance columnar format. |

## ⚡ Standalone Vector Databases

### Qdrant
An open-source, production-ready vector similarity search engine. It provides an API (gRPC and REST) to store and search vectors with complex metadata filters.
*   **GitHub**: [qdrant/qdrant](https://github.com/qdrant/qdrant)
*   **Key Strengths**:
    *   Highly optimized index construction using HNSW (Hierarchical Navigable Small World).
    *   Payload filtering: allows filtering vectors during search rather than post-search.
    *   Written in Rust, offering high performance and safety.

### Pinecone
A proprietary, fully managed cloud-native vector database.
*   **Website**: [pinecone.io](https://www.pinecone.io/)
*   **Key Strengths**:
    *   Serverless architecture: scales compute and storage independently.
    *   No clusters to manage, back up, or configure.
    *   Seamless integration with major LLM platforms (LangChain, LlamaIndex, OpenAI).

### Milvus
An open-source vector database built for high reliability and massive scale.
*   **GitHub**: [milvus-io/milvus](https://github.com/milvus-io/milvus)
*   **Key Strengths**:
    *   Decoupled architecture: splits query, data, and index nodes.
    *   Supports a wide range of index types (IVF, HNSW, ANNOY).

### Weaviate
An open-source vector database that allows you to store data objects and vector embeddings.
*   **GitHub**: [weaviate/weaviate](https://github.com/weaviate/weaviate)
*   **Key Strengths**:
    *   Can automatically connect to model APIs (OpenAI, Cohere, Hugging Face) to vectorize data on write.
    *   Strong support for hybrid search (combining BM25 keyword search and dense vector search).

## 🧩 Embedded & Relational Integrations

### pgvector
A PostgreSQL extension that enables open-source vector similarity search directly within Postgres.
*   **GitHub**: [pgvector/pgvector](https://github.com/pgvector/pgvector)
*   **Key Strengths**:
    *   No need to maintain a separate database server.
    *   Perform joint queries (e.g., filter on SQL columns, join tables, and sort by cosine distance in one SQL statement).
    *   Supports IVFFlat and HNSW indexes.

### Chroma
An AI-native open-source embedding database designed to make it easy to build LLM apps.
*   **GitHub**: [chroma-core/chroma](https://github.com/chroma-core/chroma)
*   **Key Strengths**:
    *   Can run embedded in a Python script or as a standalone Docker container.
    *   Ideal for developers who want a quick, local, memory-based database that "just works".

### LanceDB
An open-source serverless vector database written in Rust. It is built on top of the Lance columnar data format.
*   **GitHub**: [lancedb/lancedb](https://github.com/lancedb/lancedb)
*   **Key Strengths**:
    *   Serverless architecture: stores data directly in object storage (AWS S3, Google Cloud Storage, or local disk).
    *   Extremely fast disk-based vector index. Excellent for multimodal data (images, video, text).

### Faiss (Facebook AI Similarity Search)
A library for efficient similarity search and clustering of dense vectors.
*   **GitHub**: [facebookresearch/faiss](https://github.com/facebookresearch/faiss)
*   **Key Strengths**:
    *   A low-level algorithm library rather than a structured database.
    *   Contains algorithms that search sets of vectors of any size, including GPU implementations.
