# Contributing to AI Developer Resources Hub (A.I.D.)

Thank you for your interest in contributing! This repository thrives on community contributions to keep information current, high-quality, and valuable for AI developers, machine learning engineers, and research professionals.

---

## 🎯 How You Can Contribute

### 1. Suggest New Resources
- AI/ML frameworks, libraries, and agentic SDKs
- Inference and high-performance serving tools
- Vector databases, indexing strategies, and RAG components
- Evaluation, observability, and tracing platforms
- LLM optimization, quantization, and fine-tuning libraries
- AI coding assistants and local execution runtimes
- High-quality learning resources, research papers, and guides

### 2. Update Existing Information
- Missing links or typos
- Library updates or major version migrations (e.g. LangChain v0.1+ upgrades)
- Up-to-date pricing or licensing details for commercial AI APIs
- Deprecated tools or APIs

### 3. Share Event & Meetup Details
- Regional AI/ML meetups (especially in Washington DC & SFO Bay Area)
- Major conferences and workshops
- Upcoming hackathons

---

## 📝 Contribution Guidelines

### Content Quality Standards

#### For Frameworks, Tools & Libraries
- ✅ Include the official GitHub repository and website link.
- ✅ Provide a brief, objective, and accurate description of its core value proposition.
- ✅ Mention key differentiators (e.g. "SGLang: high-throughput structured output generation via runtime optimizations").
- ✅ List primary use cases.

#### For Fine-Tuning & Serving Tools
- ✅ Specify hardware requirements (e.g., minimum VRAM, GPU architectures supported if relevant).
- ✅ Note if the tool is designed for local desktop deployment (e.g., Ollama, LM Studio) or server-scale environments (e.g., vLLM, Triton).

#### For Learning Resources
- ✅ Specify the target level (beginner, intermediate, advanced).
- ✅ State if the resource is free or paid.
- ✅ Link directly to the course or material, avoiding affiliate redirects.

## 🔄 Pull Request Process

### Step 1: Fork the Repository
```bash
# Click the "Fork" button on GitHub
# Clone your fork locally
git clone https://github.com/YOUR-USERNAME/aid.git
cd aid
```

### Step 2: Create a Branch
```bash
git checkout -b add-new-agent-framework
# or
git checkout -b update-vllm-serving-docs
```

### Step 3: Make Your Changes
- Edit the relevant markdown files.
- Ensure formatting matches existing tables, lists, and headers.
- Test all links.

### Step 4: Commit Your Changes
```bash
git add .
git commit -m "Add PydanticAI to agentic frameworks list"
```

### Step 5: Push and Create Pull Request
```bash
git push origin your-branch-name
```
Then create a pull request on GitHub with:
- A clear, descriptive title.
- A brief description of what you added/updated and why it is valuable.

---

## ✅ Checklist Before Submitting

- [ ] All relative and external links are valid and active.
- [ ] Descriptions are clear, concise, and objective.
- [ ] Formatting aligns with the repository's style.
- [ ] No spelling or grammar errors.
- [ ] Commit message is clear and descriptive.

## 🚫 What Not to Include

### Avoid
- ❌ Affiliate links, referral codes, or sponsored content.
- ❌ Plagiarized summaries or marketing copy.
- ❌ Outdated, unmaintained, or abandoned repositories (typically inactive for >18 months unless foundational).
- ❌ Spam or low-quality tutorials.

## 🤝 Code of Conduct

- **Be Respectful**: Treat all contributors and reviewers with respect. Provide helpful, constructive feedback.
- **Be Collaborative**: Help others refine their pull requests and share knowledge generously.
